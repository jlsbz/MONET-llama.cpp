# 基于 llama.cpp 的 Monarch 矩阵结构化压缩、源码修改与 Android 端部署完整方案

## 0. 项目目标

本项目目标是在 `llama.cpp` 中支持一种基于 Monarch 矩阵的结构化模型压缩方案，并最终完成 Windows / WSL / Android 手机端部署验证。

与普通低比特量化不同，本方案不是简单将 dense weight 量化为 INT4 / INT8，也不是使用普通非结构化稀疏格式，而是将 LLaMA 模型中的部分 Linear 权重近似表示为 Monarch 矩阵：

```text
W ≈ P_L · L · P_R · R
```

其中：

```text
P_L / P_R：置换矩阵或置换索引
L / R：分块对角矩阵
```

在 `n = m^2` 的 square matrix 情况下，Monarch 矩阵可以将原始 dense 矩阵的参数量从：

```text
O(n^2)
```

降低到约：

```text
O(n^{3/2})
```

对于 LLaMA-2-7B 中的 `4096 × 4096` attention projection，取：

```text
block_size = 64
num_blocks = 64
```

则每个 Monarch Linear 只需要保存：

```text
L:    [64, 64, 64]
R:    [64, 64, 64]
perm: [4096]
```

而不是完整的：

```text
W: [4096, 4096]
```

# 1. 整体流程概览

完整流程分为五个阶段：

```text
阶段 1：PyTorch 侧拟合 Monarch 参数
阶段 2：Monarch 参数展开回 HF 模型，验证模型质量
阶段 3：生成 Monarch-extra GGUF，扩展模型文件格式
阶段 4：修改 llama.cpp loader / graph / ggml runtime
阶段 5：Windows + Android 手机端部署与 benchmark
```

整体路径为：

```text
HuggingFace LLaMA-2-7B
        ↓
采集 WikiText-2 activation
        ↓
拟合 q/k/v/o 的 Monarch 参数
        ↓
保存 L / R / perm
        ↓
生成 Monarch-dense HF 模型
        ↓
转换为 Monarch-dense GGUF 进行正确性验证
        ↓
生成 Monarch-extra GGUF
        ↓
修改 llama.cpp loader 读取 Monarch tensor
        ↓
修改 graph 使用 Monarch projection
        ↓
实现真正的 MonarchLinear runtime
        ↓
生成 Monarch-only GGUF
        ↓
Android 手机端部署与 benchmark
```

# 4. PyTorch 侧 Monarch 参数生成

## 4.1 目标层

当前优先处理 LLaMA-2-7B 中所有 `4096 × 4096` 的 attention projection：

```text
model.layers.i.self_attn.q_proj
model.layers.i.self_attn.k_proj
model.layers.i.self_attn.v_proj
model.layers.i.self_attn.o_proj
```

暂时不处理 MLP：

```text
model.layers.i.mlp.gate_proj: 11008 × 4096
model.layers.i.mlp.up_proj:   11008 × 4096
model.layers.i.mlp.down_proj: 4096 × 11008
```

原因是 MLP 矩阵是 rectangular，需要后续设计 rectangular Monarch 或分块策略。

---

## 4.2 拟合目标

对每个目标 Linear 层，采集真实输入 activation：

```text
X = input activation of Linear layer
```

拟合目标为：

```text
MonarchLinear(X) ≈ DenseLinear(X)
```

而不是仅拟合：

```text
W_monarch ≈ W_dense
```

这样可以让 Monarch 近似更贴近真实推理分布。

---

## 4.3 输出文件

每个目标层保存一个 `.pt` 文件：

```text
model__layers__0__self_attn__q_proj.pt
model__layers__0__self_attn__k_proj.pt
model__layers__0__self_attn__v_proj.pt
model__layers__0__self_attn__o_proj.pt
...
```

每个文件包含：

```python
{
    "layer_name": "model.layers.0.self_attn.q_proj",
    "in_features": 4096,
    "out_features": 4096,
    "block_size": 64,
    "num_blocks": 64,
    "L": Tensor[64, 64, 64],
    "R": Tensor[64, 64, 64],
    "perm": Tensor[4096],
    "metrics": {
        "mse": ...,
        "rel_mse": ...,
        "mae": ...,
        "rel_mae": ...
    }
}
```

# 5. Monarch-dense HF 模型验证

## 5.1 目的

这一步将 Monarch 参数重新展开为 dense weight，替换原始 HuggingFace 模型中的 `q/k/v/o` 权重。

该步骤的目的不是压缩，而是验证：

```text
1. 拟合得到的 Monarch 参数是否合理；
2. 展开后的权重是否能让模型正常生成；
3. 模型输出是否明显崩坏；
4. 后续 llama.cpp 修改前先排除拟合质量问题。
```

---

## 5.2 重构公式

PyTorch 中的 Monarch forward 为：

```text
x1 = x @ R_blockdiag
x2 = x1[:, perm]
y  = x2 @ L_blockdiag
```

普通 Linear 为：

```text
y = x @ W^T
```

因此重构：

```text
W^T = R_dense @ P @ L_dense
W   = (R_dense @ P @ L_dense)^T
```

---

## 10.2 插入 Monarch tensor 认领逻辑

在 `layer.wo = ...` 后面、`layer.wo_b = ...` 前面插入：

```cpp
{
    const int64_t monarch_block_size = 64;

    if (n_embd % monarch_block_size == 0) {
        const int64_t monarch_num_blocks = n_embd / monarch_block_size;

        auto create_monarch_weight = [&](llama_monarch_weight & mw, const char * base) {
            const std::string prefix = "blk." + std::to_string(i) + "." + std::string(base);

            const std::string name_l    = prefix + ".monarch_l";
            const std::string name_r    = prefix + ".monarch_r";
            const std::string name_perm = prefix + ".monarch_perm";

            mw.l = create_tensor(
                name_l,
                {monarch_block_size, monarch_block_size, monarch_num_blocks},
                TENSOR_NOT_REQUIRED
            );

            mw.r = create_tensor(
                name_r,
                {monarch_block_size, monarch_block_size, monarch_num_blocks},
                TENSOR_NOT_REQUIRED
            );

            mw.perm = create_tensor(
                name_perm,
                {n_embd},
                TENSOR_NOT_REQUIRED
            );

            mw.enabled = (mw.l != nullptr && mw.r != nullptr && mw.perm != nullptr);
            mw.block_size = (int32_t) monarch_block_size;
            mw.num_blocks = (int32_t) monarch_num_blocks;

            if (mw.enabled) {
                LLAMA_LOG_INFO(
                    "monarch: layer %d %s enabled, block_size=%lld, num_blocks=%lld\n",
                    i,
                    base,
                    (long long) monarch_block_size,
                    (long long) monarch_num_blocks
                );
            }
        };

        create_monarch_weight(layer.wq_monarch, "attn_q");
        create_monarch_weight(layer.wk_monarch, "attn_k");
        create_monarch_weight(layer.wv_monarch, "attn_v");
        create_monarch_weight(layer.wo_monarch, "attn_output");
    }
}
```

## 12.3 CPU reference 实现

单 token decode 的逻辑：

```cpp
// tmp1 = x @ R_blockdiag
for b in range(num_blocks):
    for j in range(block_size):
        tmp1[b * B + j] = 0
        for i in range(block_size):
            tmp1[b * B + j] += x[b * B + i] * R[b][i][j]

// tmp2 = tmp1[perm]
for j in range(hidden):
    tmp2[j] = tmp1[perm[j]]

// y = tmp2 @ L_blockdiag
for b in range(num_blocks):
    for j in range(block_size):
        y[b * B + j] = 0
        for i in range(block_size):
            y[b * B + j] += tmp2[b * B + i] * L[b][i][j]
```

---

## 12.4 ARM NEON 优化方向

手机端真正部署时，需要针对 ARM CPU 优化：

```text
1. block_size = 64 的小矩阵乘法
2. 连续内存布局 [num_blocks, 64, 64]
3. 避免显式 materialize permutation
4. 将 permutation fuse 到第二个 blockdiag matmul
5. 使用 fp16 / int8 L/R
6. 多线程按 block 或 token 并行
```

# 13. Monarch-only GGUF 规划

若不做进一步优化，则 `monarch-extra.gguf` 同时保存 dense weight 和 Monarch tensor，因此文件更大。

目标是：

```text
llama2-7b-monarch-only.gguf
```

该文件删除被替换层的 dense weight，只保留：

```text
blk.i.attn_q.monarch_l
blk.i.attn_q.monarch_r
blk.i.attn_q.monarch_perm
```

此时才会真正减少模型大小。因此需要进行如下改动：

---

## 13.1 Converter 修改

在 `convert_hf_to_gguf_monarch.py` 中增加参数：

```text
--monarch-only
```

如果启用：

```text
1. 对已存在 Monarch 参数的 q/k/v/o 层，不写原 dense weight；
2. 只写 monarch_l / monarch_r / monarch_perm；
3. 对没有 Monarch 参数的层，仍然写 dense weight。
```

---

## 13.2 Loader 修改

Monarch-only 模型中，部分 dense tensor 不存在。因此 loader 需要允许：

```text
attn_q.weight 缺失，但 attn_q.monarch_l/r/perm 存在
```

也就是说：

```text
if dense weight exists:
    load dense
if Monarch tensor exists:
    load Monarch
if both exist:
    allow fallback or debug
if neither exists:
    error
```

当前 `create_tensor_qkv(...)` 默认会强制加载 dense q/k/v。修改为：

```text
q/k/v/o dense weight 可选
Monarch weight 可选
二者至少存在一种
```

# 14. Android 端部署修改

## 14.1 当前 Android 部署方式

基础 Android 部署路径为：

```text
Windows / WSL 生成 GGUF
        ↓
Windows 使用 Android NDK 编译 llama.cpp
        ↓
adb push 可执行文件和 GGUF
        ↓
手机端运行 llama-cli / llama-bench
```

该流程与普通 llama.cpp Android 部署类似。此前 Android 端部署说明中已经包含 WSL、Windows、Android NDK、ADB、Termux、GGUF、Q4_K_M 和 benchmark 等步骤。

---

## 14.2 Android 端需要同步的源码修改

当 `llama.cpp-monarch` 修改完成后，Android 端不能再使用原始 `llama.cpp-baseline` 编译。

必须使用：

```text
llama.cpp-monarch/
```

因为其中包含：

```text
1. llama_layer 中的 Monarch tensor 字段
2. LLaMA loader 中的 Monarch tensor 创建逻辑
3. graph 中的 Monarch projection 逻辑
4. ggml 中的 MonarchLinear op
5. CPU / ARM backend 实现
```

---

## 14.3 Android 编译命令

Windows PowerShell 中：

```powershell
$env:ANDROID_NDK="C:\Users\<用户名>\AppData\Local\Android\Sdk\ndk\<NDK版本号>"

cd $env:USERPROFILE\llama.cpp-monarch

Remove-Item -Recurse -Force build-android -ErrorAction SilentlyContinue

cmake -S . -B build-android -G "Ninja" `
  -DCMAKE_BUILD_TYPE=Release `
  -DBUILD_SHARED_LIBS=OFF `
  -DGGML_OPENMP=OFF `
  -DCMAKE_TOOLCHAIN_FILE="$env:ANDROID_NDK\build\cmake\android.toolchain.cmake" `
  -DANDROID_ABI=arm64-v8a `
  -DANDROID_PLATFORM=android-28

cmake --build build-android -j4
```

---

## 14.4 为什么源码修改后必须重新编译 Android 版

如果只换模型文件，例如：

```text
llama2-7b-q4_k_m.gguf
→ llama2-7b-monarch-extra.gguf
```

理论上只需要重新 push 模型。

但如果修改了以下内容：

```text
src/llama-model.h
src/llama-model.cpp
src/models/llama.cpp
ggml op
ggml CPU backend
```

则必须重新编译 Android 版：

```text
llama-cli
llama-bench
```

否则手机端二进制不包含 Monarch 支持。
