# Android 手机端 LLaMA 部署教程（llama.cpp）

本项目用于演示如何基于 `llama.cpp` 将 LLaMA 系列模型部署到 Android 手机端，并进行移动端推理与 benchmark 测试。

当前流程支持：

* Windows + WSL2 开发环境
* GGUF 模型转换
* 模型量化（Q4/Q5/Q6）
* Android NDK 交叉编译
* ADB 部署
* Termux 本地运行
* 手机端 benchmark
* PC 与手机端同步推理展示

---

# 一、整体流程

整体部署路径如下：

```text
HuggingFace 模型
        ↓
GGUF 转换
        ↓
模型量化
        ↓
Android NDK 编译 llama.cpp
        ↓
ADB push / Termux
        ↓
手机端推理
```

其中：

* WSL 负责模型处理
* Windows 负责 Android 交叉编译
* Android 手机负责实际推理运行

# 三、编译 llama.cpp（WSL）

克隆：

```bash
git clone https://github.com/ggerganov/llama.cpp
cd llama.cpp
```

创建 Python 虚拟环境：

```bash
python3 -m venv .venv
source .venv/bin/activate
```

安装依赖：

```bash
pip install -U pip
pip install -r requirements.txt
```

编译：

```bash
cmake -S . -B build -G Ninja -DCMAKE_BUILD_TYPE=Release

cmake --build build -j4
```

生成：

```text
build/bin/llama-cli
build/bin/llama-bench
build/bin/llama-quantize
```

---

# 四、HF 模型转换 GGUF

假设模型位于：

```text
/mnt/d/models/llama2-7b-hf
```

执行：

```bash
python3 convert_hf_to_gguf.py \
  /mnt/d/models/llama2-7b-hf \
  --outtype f16 \
  --outfile llama2-7b-f16.gguf
```

---

# 五、模型量化

推荐：

```text
Q4_K_M
```

执行：

```bash
./build/bin/llama-quantize \
  llama2-7b-f16.gguf \
  llama2-7b-q4_k_m.gguf \
  Q4_K_M
```

# 六、Android NDK 交叉编译

Windows PowerShell：

```powershell
$env:ANDROID_NDK="C:\Users\用户名\AppData\Local\Android\Sdk\ndk\版本号"
```

编译 Android 版本：

```powershell
cmake -S . -B build-android -G "Ninja" `
  -DCMAKE_BUILD_TYPE=Release `
  -DBUILD_SHARED_LIBS=OFF `
  -DGGML_OPENMP=OFF `
  -DCMAKE_TOOLCHAIN_FILE="$env:ANDROID_NDK\build\cmake\android.toolchain.cmake" `
  -DANDROID_ABI=arm64-v8a `
  -DANDROID_PLATFORM=android-28

cmake --build build-android -j4
```

---

# 七、ADB 部署

创建目录：

```powershell
adb shell "mkdir -p /data/local/tmp/llama"
```

推送：

```powershell
adb push llama-cli /data/local/tmp/llama/
adb push llama2-7b-q4_k_m.gguf /data/local/tmp/llama/
```

赋予权限：

```powershell
adb shell "chmod +x /data/local/tmp/llama/llama-cli"
```

---

# 八、手机端运行

进入 shell：

```powershell
adb shell
```

运行：

```bash
cd /data/local/tmp/llama

./llama-cli \
  -m llama2-7b-q4_k_m.gguf \
  -p "Hello" \
  -n 64 \
  -t 8
```

# 九、Benchmark 测试

推荐 benchmark：

```bash
./llama-bench \
  -m llama2-7b-q4_k_m.gguf \
  -p 128 \
  -n 128 \
  -t 8
```

# 十二、常见问题

## libomp.so not found

关闭 OpenMP：

```text
-DGGML_OPENMP=OFF
```

---

## libmtmd.so not found

使用：

```text
-DBUILD_SHARED_LIBS=OFF
```

---

## WSL 编译 OOM

配置：

```text
.wslconfig
```

增加：

```text
memory=12GB
swap=16GB
```

---

## Termux 速度极慢

原因：

* 后台限频
* battery optimization
* 单线程

建议：

```bash
termux-wake-lock
-t 8
```
